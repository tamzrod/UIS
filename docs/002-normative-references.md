# Normative References

## 2.1 General

The following documents are referred to in the text in such a way that some or all of their content constitutes requirements of this document. For dated references, only the edition cited applies. For undated references, the latest edition of the referenced document (including any amendments) applies.

## 2.2 Referenced Standards

| Reference | Title | Relationship |

| ISO 704 | Terminology work — Principles and methods | UIS terminology practices follow ISO 704 guidance |
| ISO 8601 | Date and time — Representations for information interchange | UIS may reference ISO 8601 for temporal representations |
| IEC 61360 | Standard data element types with associated classification scheme | UIS information objects may reference IEC 61360 data types |

## 2.4 Conformance References

Conformance to UIS is defined in [Clause 11](011-conformance.md). Normative requirements within UIS use the following conventions:

- **shall**: Indicates a mandatory requirement
- **should**: Indicates a recommendation or preferred practice
- **may**: Indicates a permitted option or allowed behavior
- **can**: Indicates a capability or possibility

Interpretation of these keywords follows [RFC 8174](https://www.rfc-editor.org/rfc/rfc8174).
